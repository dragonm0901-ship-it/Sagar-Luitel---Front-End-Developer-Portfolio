Chapter 1: Getting Started with HTML
1.1 Welcome to the Course
Welcome to the complete HTML and CSS course. In this course, we are going to learn how to build websites from a beginner to a professional level. By the end of this journey, we are going to build a clone of YouTube.com. You don't need any previous coding or technical experience; this is designed to be your first step to becoming a software engineer.
1.2 Setting Up Your Tools
Before we begin, we need two pieces of software:
1. A Web Browser: This lets us view websites on the internet and view the websites we create. We will be using Google Chrome.
2. A Code Editor: This helps us write our HTML and CSS code efficiently. The most popular code editor for web development is Visual Studio Code (VS Code).
Action Step: Go to google.com, search for Google Chrome and VS Code, and install them on your computer.
1.3 What is HTML?
HTML stands for Hypertext Markup Language. You don't have to worry about the exact definition right now. Just know that it is a tool we use to create websites. Every website, from YouTube to Amazon, uses a combination of HTML and CSS.
The easiest way to understand HTML is that we are simply giving instructions to a computer. We are telling the computer what to do step-by-step, and the computer follows those instructions to create our website.
1.4 Your First HTML File
Let's jump in.
1. Create a new folder on your computer named HTML-CSS-course.
2. Open VS Code, click File > Open, and select the folder you just created.
3. Click the "New File" icon and name your file website.html.
Note: The file must end with .html. This tells the computer that this file contains HTML code, not just plain text.
Inside website.html, type your first instruction:
HTML
<button>hello</button>


To view this, go to your folder on your computer, right-click website.html, and select Open with > Google Chrome. You will see a physical button on a white page with the word "hello" inside it. The computer followed your exact instruction!
Let's add another instruction on a new line:
HTML
<p>paragraph of text</p>


Save your file, go to Chrome, and click Refresh. You will now see your text beneath the button. The computer reads HTML from top to bottom. If you put the <p> code above the <button> code, the paragraph will appear on top.
1.5 HTML Terminology and Syntax
* Elements: Each of the things we display on the page (a button, a paragraph) is called an HTML Element.
* Syntax: Syntax represents the rules for how a coding language should be written. It is like grammar in English. However, if you have bad English grammar, people can still understand you. If you break HTML syntax, the computer will not understand you.
The Anatomy of an HTML Tag: Elements consist of tags.
* Opening Tag: <button> (Tells the computer what we are starting to create).
* Content: hello (The text inside).
* Closing Tag: </button> (The exact same as the opening tag, but with a forward slash / in front of the name to tell the computer the element has ended).
Experiment: If you delete the closing > on your opening button tag and refresh your browser, your button will disappear. You broke the rules of syntax!
1.6 Links and Attributes
Let's learn a new element: the Anchor element (<a>), which creates a link to another website.
HTML
<a>Link to YouTube</a>


If you save and click this on your webpage, nothing happens. It is a link without a destination. We must add an Attribute.
Attributes modify how an element behaves. To add a destination, we use the href attribute inside the opening tag, separated by a space:
HTML
<a href="https://www.youtube.com/">Link to YouTube</a>


* Attribute Name: href (Tells the computer what we are modifying).
* Attribute Value: "https://www.youtube.com/" (Tells the computer what we are modifying it to. It must be inside double quotes).
Let's add a second attribute, separated by a space, to make the link open in a new tab:
HTML
<a href="https://www.youtube.com/" target="_blank">Link to YouTube</a>


1.7 The Quirks of HTML Spacing
In HTML, extra spaces and new lines are ignored by the browser. If you type: <button>hello</button> <a>Link</a> The browser will display them with only one space between them.
This quirk is actually helpful! It means we can use new lines and spaces (indentation) to cleanly organize our code without messing up how the website looks. In HTML and CSS, the standard is to use 2 spaces per indent. (You can change this in VS Code by clicking "Indent using spaces" in the bottom right toolbar and changing it from 4 to 2).

Chapter 2: Styling Basics with CSS
2.1 What is CSS?
CSS stands for Cascading Style Sheets. While HTML creates the elements, CSS changes their appearance.
Create a new file called buttons.html. Let's create a standard HTML button:
HTML
<button>SUBSCRIBE</button>


To write CSS, we must create a special HTML element called the <style> element. This element doesn't visibly appear on the page; its sole purpose is to hold our CSS instructions.
HTML
<style>
  button {
    background-color: red;
    color: white;
    border: none;
  }
</style>


2.2 CSS Syntax Rules
Let's break down the CSS code above:
* Selector (button): Tells the computer which HTML elements on the page we are targeting. Here, we are targeting all <button> elements.
* Curly Braces { }: Contain the styling rules.
* Property (background-color): Tells the computer what feature we are changing.
* Value (red): Tells the computer what we are changing it to.
* Punctuation: Properties and values are separated by a colon (:). Every CSS instruction must end with a semicolon (;), just like ending an English sentence with a period.
2.3 Sizing and Trial & Error
Let's make our button look like the official YouTube subscribe button.
CSS
 height: 36px;
  width: 105px;


Pixels (px) are the standard unit of measurement in the digital world. You don't need to inherently know exactly how big 36px is. CSS involves a lot of trial and error. You try 50px, see it's too big, shrink it to 30px, see it's too small, and settle on 36px.
2.4 RGB Colors and Border Radius
"Red" is not a precise color. To match YouTube perfectly, we use RGB (Red, Green, Blue) values. Every digital color is a mix of Red, Green, and Blue, with values ranging from 0 to 255.
* rgb(255, 255, 255) is pure white.
* rgb(0, 0, 0) is pure black.
* rgb(200, 0, 0) is a specific shade of red. (In VS Code, hover over the color to bring up a color picker!)
Let's finish the YouTube button by rounding the corners and changing our mouse to a pointer hand when we hover over it:
CSS
 border-radius: 2px;
  cursor: pointer;


2.5 The Class Attribute (Labeling Elements)
What if we add a second button for "JOIN"?
HTML
<button>JOIN</button>


Because our CSS selector is button, both buttons turn red! To style them differently, we must label them using the class attribute.
HTML
<button class="subscribe-button">SUBSCRIBE</button>
<button class="join-button">JOIN</button>


Now, in our CSS, instead of typing button, we type a period (.) followed by the class name to target only elements with that specific label:
CSS
.subscribe-button {
  background-color: rgb(200, 0, 0);
  /* other styles... */
}


.join-button {
  background-color: white;
  border-color: rgb(41, 98, 255);
  border-style: solid;
  border-width: 1px;
  color: rgb(41, 98, 255);
  /* ... */
}


By creating elements in HTML and carefully targeting them with CSS classes, we can perfectly recreate the visual style of any famous website.
Here is the continuation of your comprehensive manual, picking up right where we left off.

Chapter 3: Intermediate CSS
3.1 Pseudo-Classes: :hover and :active
For most buttons you see on the internet, their style changes slightly when you hover over them with a mouse—some get darker, some get lighter. To recreate this, we need to learn about pseudo-classes, which add extra styles in specific situations.
Let's go back to our .subscribe-button class from Chapter 2. We can create a new block of CSS right below it using the :hover pseudo-class:
CSS
.subscribe-button:hover {
  background-color: green;
}


If we are not hovering, the base styles apply (the button is red). If we are hovering, the computer applies the base styles plus the :hover styles, turning the button green.
Another useful pseudo-class is :active, which activates only while you are actively clicking down on the element:
CSS
.subscribe-button:active {
  background-color: blue;
}


3.2 Opacity
Instead of changing the background color to green, professional websites often just make the button slightly faded when hovered. We achieve this with the opacity property.
Opacity tells the computer how "see-through" an element is. It takes a value between 0 and 1:
* 1 is completely solid (the default).
* 0.8 is slightly see-through.
* 0 is completely invisible.
Let's update our hover and active states to use opacity:
CSS
.subscribe-button:hover {
  opacity: 0.8;
}


.subscribe-button:active {
  opacity: 0.5;
}


Now, hovering makes the button slightly lighter, and clicking makes it even lighter!
3.3 Transitions
If you look at modern websites, hover effects do not happen instantly; they fade in smoothly. We can achieve this using the transition property.
The transition property requires two values:
1. What we want to transition (e.g., opacity).
2. How long the transition should take (e.g., 0.15s for 0.15 seconds).
Crucial Rule: You must put the transition property in the base CSS block, not the :hover block. If you put it in the hover block, the transition only works when your mouse goes on the button, but it will snap back instantly when your mouse leaves.
CSS
.subscribe-button {
  background-color: rgb(200, 0, 0);
  color: white;
  border: none;
  border-radius: 2px;
  cursor: pointer;
  transition: opacity 0.15s; /* Smoothly transitions the opacity */
}


You can transition multiple properties at once by separating them with a comma. For our .join-button, we might want to invert the background and text colors when hovered:
CSS
.join-button {
  /* ...base styles... */
  transition: background-color 0.15s, color 0.15s;
}


3.4 Box Shadows and RGBA
Let's add a realistic drop shadow to a button. The box-shadow property takes four distinct values:
1. Horizontal Position: How far right the shadow moves (e.g., 5px).
2. Vertical Position: How far down the shadow moves (e.g., 5px).
3. Blur: How fuzzy the shadow looks (e.g., 10px).
4. Color: The color of the shadow.
To make a shadow look natural, it needs to be very faint. Pure black is too harsh. We will use a new color measurement called RGBA. It works exactly like RGB, but the 'A' stands for "Alpha"—which is simply the opacity of that specific color!
CSS
.tweet-button {
  /* ...base styles... */
  box-shadow: 5px 5px 10px rgba(0, 0, 0, 0.15);
}


This creates a shadow pushed 5px right, 5px down, with a 10px blur, using pure black (0, 0, 0) that is incredibly faint (0.15 opacity).

Chapter 4: The CSS Box Model
4.1 Chrome DevTools and Hex Colors
Before learning the Box Model, we need to learn how to steal exact styles from professional websites. Open any website in Google Chrome, right-click an empty area, and click Inspect. This opens the Chrome DevTools.
1. Click the small "Pointer/Arrow" icon in the top left of the DevTools window.
2. Hover over any element on the website (like a button) and click it.
3. Look at the "Computed" tab in the DevTools menu.
This reveals every final CSS style applied to that element! You might notice background colors written like #FF0000. This is a Hex Code, which is just another way to write RGB values. You can copy these hex codes directly into your CSS or use a Google "Hex to RGB converter" to keep your code consistent.
4.2 Introducing the Box Model
Every single element in HTML is treated as a rectangular box. The CSS Box Model dictates how much physical space an element takes up on the page. It consists of four layers:
1. Content: The actual text or image (the innermost core).
2. Padding: The empty space inside the element, between the content and the border.
3. Border: The physical line wrapping around the padding and content.
4. Margin: The invisible empty space outside the element, separating it from other elements.
4.3 The Problem with Fixed Widths
In Chapter 2, we forced our button to be exactly 36px tall and 105px wide. This is actually a bad practice!
Experiment: If you change the text inside your HTML from <button>JOIN</button> to <button>JOIN MY CHANNEL</button>, the text will overflow right out of the physical button boundaries because you rigidly locked the width.
4.4 Fixing Overflow with Padding
By default, a button automatically adjusts its size based on the text inside it. Instead of forcing a rigid height and width, we should add Padding to organically pad the inside of the button.
Delete height and width from your CSS, and replace them with this:
CSS
.subscribe-button {
  padding-top: 10px;
  padding-bottom: 10px;
  padding-left: 16px;
  padding-right: 16px;
}


Now, if you change the text to "SUBSCRIBE TO MY CHANNEL", the button will organically grow to fit the text while maintaining that perfect 16px of breathing room on the sides!
4.5 Margins and Vertical Alignment
To push two buttons apart, we add space to the outside using margin:
CSS
.subscribe-button {
  margin-right: 8px; /* Adds 8px of invisible space to the right */
}


The Vertical Alignment Quirk: When you put buttons next to each other, you might notice one sits slightly lower than the other. Browsers historically align elements based on their text baseline. To force buttons to perfectly align at their physical top edges, add this to their CSS:
CSS
 vertical-align: top;


The Border Math Quirk: If your .join-button has a 1px solid border, but your .subscribe-button has no border, the Join button will actually be 2px taller overall (1px top border + 1px bottom border). To make them identical heights, you must reduce the padding on the bordered button by exactly 1px:
CSS
.join-button {
  padding-top: 9px; /* Reduced from 10px to account for the 1px border */
  padding-bottom: 9px;
  /* ... */
}

Chapter 5: Text Styling and Specificity
5.1 Basic Text Properties
For most of the course so far, we have worked with buttons. Now, let's learn how to style text using the paragraph element (<p>). By default, web browsers display text in a font called Times New Roman. We can change this and much more using CSS.
Font Family: Changes the typeface.
CSS
.video-title {
  font-family: Arial;
}
* * Font Size: Changes how large the text is (e.g., font-size: 18px;).
* Font Weight: Changes the thickness. font-weight: bold; makes text bold.
* Font Style: Changes the slant. font-style: italic; makes text italicized.
* Text Align: Centers text or aligns it to the left/right (e.g., text-align: center;).
5.2 Line Height and Text Wrapping
If you have a long title, it will normally stretch all the way across the page. If you want the text to wrap around to a second line, you must set a rigid width on the paragraph:
CSS
.video-title {
  width: 300px;
}


Once your text drops to a second line, you might notice the lines feel too close together. You can adjust the vertical breathing room between lines of text using the line-height property (e.g., line-height: 24px;).
5.3 HTML Entities
Sometimes you need to type special characters, like a middle dot (·), a checkmark (✓), or a greater-than symbol (>). If you just type a > symbol in your HTML, the browser might confuse it with a closing tag and break your code.
To safely display these, we use HTML Entities—special codes that the browser translates into symbols.
* Middle Dot: &#183;
* Checkmark: &#10003;
Action Step: Whenever you need a special symbol, just search Google for "HTML entity [symbol name]" and paste the code into your text.
5.4 The Default Margin Trap
If you put two <p> elements on top of each other, you will notice a large gap between them, even if you haven't written any CSS margin rules.
Why? Paragraph elements come with invisible default margins baked into the browser. To gain total control over your design, you must first reset these defaults to zero, and then apply your own precise spacing:
CSS
.video-stats {
  margin-top: 0;
  margin-bottom: 20px; /* Setting our exact desired spacing */
}


5.5 Inline Text Elements (<span>)
What if you want to make just one single word inside a paragraph red, or underline a specific phrase when hovered? You cannot target the whole <p> element. You must wrap the specific word in a Text Element.
Text elements exist inside a line of text without breaking it onto a new line.
* <strong>: Automatically makes the wrapped text bold.
* <u>: Automatically underlines the wrapped text.
* <span>: The most powerful text element. It comes with zero default styles. It is a blank canvas that you can label with a class and style perfectly in your CSS.
HTML
<p>Shop early for the <span class="shop-link">best selection</span></p>


CSS
.shop-link:hover {
  text-decoration: underline;
  cursor: pointer;
}


5.6 CSS Specificity (The Priority Rule)
Imagine you have two CSS rules fighting each other:
1. A rule targeting all paragraphs (p { font-family: Arial; })
2. A rule targeting a specific class (.video-title { font-family: Roboto; })
Which one wins? The browser uses a priority system called Specificity. A class selector (like .video-title) is considered more specific than a broad element selector (like p). Therefore, the class rule overrides the element rule, regardless of what order they are written in your CSS file.

Chapter 6: Proper HTML Structure
6.1 The Invisible Skeleton
Up until now, we have just been throwing <button> and <p> tags directly into a blank file. The browser is smart enough to guess what we mean, but we are missing out on powerful features. Every professional HTML file must have a strict, nested structure.
Delete your old code and type exactly this:
HTML
<!DOCTYPE html>
<html>
  <head>
    <title>My First Website</title>
  </head>
  <body>
    </body>
</html>


* <!DOCTYPE html>: Not technically an element. It is a special declaration telling the browser to use the modern HTML5 standard.
* <html>: The root container for the entire webpage.
* <head>: Contains all the "invisible" elements, such as the <title> (which changes the text in your browser tab) and links to your CSS.
* <body>: Contains absolutely everything that is visibly displayed on the page.
6.2 Live Server
Now that we have the proper structure, we can install a magical VS Code extension called Live Server. Instead of constantly clicking "Refresh" in Chrome every time you change your code, you can right-click your HTML file and select "Open with Live Server". Every time you hit save in your code editor, the browser will instantly and automatically update.
6.3 Separating HTML and CSS
Mixing HTML and CSS in one file gets messy very fast. The professional standard is to separate them.
1. Create a new file called text.css.
2. Cut all your CSS code from inside the <style> tags and paste it into text.css.
3. Delete the empty <style> tags from your HTML.
Now, your HTML file is broken because it doesn't know where the styles went. We must link them back together inside the <head> element using a Void Element.
6.4 Void Elements and File Paths
A void element is an HTML element that does not have a closing tag. The <link> element is a void element used to connect files.
HTML
<head>
  <link rel="stylesheet" href="text.css">
</head>


* rel="stylesheet": Tells the browser the relationship of the file (it's a style sheet).
* href="text.css": Tells the browser the path to find the file.
File Paths: If your CSS file is sitting right next to your HTML file, you just type its name. But what if you put your CSS inside a folder named styles to stay organized? You must update the path to navigate into that folder using a forward slash: href="styles/text.css".
6.5 Loading Google Fonts
The final benefit of a proper <head> structure is the ability to load beautiful, custom fonts from the internet instead of relying on boring default fonts like Arial.
1. Go to fonts.google.com.
2. Search for a font (e.g., "Roboto").
3. Select the weights you need (e.g., Regular 400, Medium 500, Bold 700).
4. Copy the block of <link> tags provided by Google.
5. Paste those tags directly into your HTML <head>.
6. In your CSS, you can now write font-family: Roboto; and the browser will beautifully render the new font!
Chapter 7: Images and Text Boxes
7.1 Setting Up the Final Project
It is time to start building our final project: a professional clone of YouTube.com.
1. Create a new folder on your computer named intro-to-html.
2. Move all your previous practice files into a separate folder to keep things clean.
3. Inside intro-to-html, create a new file named youtube.html.
4. Set up the proper HTML structure (<!DOCTYPE html>, <html>, <head>, <body>).
5. Open it with Live Server.
7.2 The Image Element (<img>)
To build YouTube, we need video thumbnails. You can download sample thumbnails from the instructor's reference site by right-clicking them, selecting "Inspect," and finding the image URL in the Chrome DevTools. Save them into a new folder named thumbnails inside your project folder.
To display an image on your website, you use the Image Element. Like the <link> element, the <img> element is a Void Element—it does not have a closing tag.
HTML
<img src="thumbnails/thumbnail-1.webp">


* The src Attribute: Short for "source." This tells the computer exactly where to find the image file. It uses the exact same file path rules we learned when linking CSS files.
7.3 Image Sizing and object-fit
When you load an image, it will display at its original, massive resolution. We need to resize it using CSS.
CSS
.thumbnail {
  width: 300px;
}


The Aspect Ratio Quirk: You might notice we only set the width. When you set the width of an image, the browser automatically calculates and adjusts the height to maintain the image's original dimensions (its aspect ratio). It prevents the image from looking squished or stretched.
If you force both a width and a height (e.g., width: 300px; height: 600px;), the image will lose its shape and stretch terribly. To fix a forced shape, CSS provides image-specific properties:
* object-fit: cover;: Tells the image to completely cover the 300x600 area, even if it has to chop off the edges of the image to keep its shape.
* object-fit: contain;: Tells the image to shrink itself until the entire picture is fully contained and visible within the 300x600 area, leaving empty space if the aspect ratios don't match.
* object-position:: Lets you shift which part of the image is visible (e.g., object-position: left; or bottom).
For our YouTube clone, we will simply set width: 300px; and let the height auto-adjust.
7.4 Text Boxes (<input>)
At the top of YouTube, there is a search bar. We create text boxes using the Input element (<input>), which is another void element.
HTML
<input type="text" placeholder="Search">


* type="text": Tells the browser to render a standard text-entry box. (If you change this to type="checkbox", it literally transforms into a clickable checkbox!)
* placeholder="Search": This attribute puts a faint gray label inside the text box that automatically disappears the moment the user starts typing.
You can style the <input> element exactly like a button. You can give it a class (like .search-bar) and use CSS to change its font-size, margin, padding, and border-radius.

Chapter 8: The Display Property
8.1 The Layout Mystery
If you put your <input> search bar and your <img> thumbnail into your HTML, you will notice something strange: they sit side-by-side on the same line. However, if you add three <p> elements below them for the video title, channel name, and view count, those paragraphs will rigidly stack on top of each other, one per line. Why do images and text boxes sit side-by-side, but paragraphs stack?
This happens because of the CSS display property.
8.2 The Three Types of Elements
In HTML, elements fundamentally behave in one of three ways:
1. Block Elements: By default, elements like <p>, <div>, and <header> are Block elements. A block element fiercely claims the entire horizontal line on the webpage. Even if you use CSS to make a paragraph's width only 300px, it will still physically block any other element from sitting next to it.
2. Inline-Block Elements: Elements like <img>, <button>, and <input> are Inline-Block elements. They only take up exactly as much width as they physically need. Because they don't claim the whole line, other Inline-Block elements can comfortably sit right next to them.
3. Inline Elements: Elements like <strong>, <u>, and <span>. These are strictly text-level elements that exist inside a line of text.
8.3 Controlling the Display Property
We are not stuck with these defaults. We can use CSS to manipulate how elements behave, forcing them to sit side-by-side or forcing them to take up their own lines.
If we want our video author and video stats paragraphs to sit side-by-side, we simply change their display property:
CSS
.video-author,
.video-stats {
  display: inline-block;
}


(Note the comma syntax above! By placing a comma between .video-author and .video-stats, we can apply the exact same CSS rule to multiple classes at once, saving us from writing duplicate code).
Now, instead of taking up the whole line, those paragraphs only take up as much space as their text requires, allowing them to sit next to each other.
Conversely, if we want our YouTube search bar to have its own dedicated row at the very top of the screen, we can force it to behave like a block:
CSS
.search-bar {
  display: block;
}


By mastering the display property, you gain the power to manually control whether elements stack vertically or align horizontally!
Chapter 9: The Div Element & Nested Layouts
9.1 The Most Important Element: The Div
If you inspect the code of professional websites like Twitter or Instagram, you will notice they are built almost entirely out of an element called <div>.
What is a <div>? It stands for "Division," but the easiest way to understand it is that a div is just a box.
By itself, a <div> does absolutely nothing visually. If you type <div>This is a div</div>, it behaves exactly like a paragraph. It is a Block element by default, meaning it takes up the entire horizontal line. However, its true power is that it acts as a container. You can put paragraphs, buttons, images, and even other divs inside of a div.
9.2 The Power of Containers
Why do we need containers? In Chapter 8, we learned that block elements take up the entire line on the page. However, the exact rule is that block elements take up the entire line in their container.
If you put a title, an author name, and a view count inside a <div> container, and then use CSS to restrict that container's width to 300px, those block elements are now forced to stay within that 300px box. They will stack neatly on top of each other inside the box, leaving the rest of the webpage empty.
Because the <div> itself is a block element, we can use CSS to change its display property:
CSS
.video-preview {
  display: inline-block;
}


Now, you can put two entire 300px <div> containers side-by-side! This is the secret to building the grid of videos on YouTube.
9.3 The Nested Layouts Technique
This is arguably the most important lesson in web development. Professional web design boils down to a technique called Nested Layouts. You can recreate almost any website in the world by breaking it down into a combination of two layouts:
1. Vertical Layouts: Elements stacked on top of each other.
2. Horizontal Layouts: Elements placed side-by-side.
Let's break down a YouTube video preview card using this technique:
* Layer 1 (Vertical): The entire card is a vertical layout consisting of two parts: the thumbnail image on top, and the video info row on the bottom.
* Layer 2 (Horizontal): Look closer at the video info row on the bottom. It is a horizontal layout consisting of two parts: the channel profile picture on the left, and the text data on the right.
* Layer 3 (Vertical): Look closer at the text data on the right. It is a vertical layout consisting of the video title on top, the author in the middle, and the stats on the bottom.
We have layouts inside of layouts inside of layouts.
9.4 Coding the Nested Layout
To build this in HTML, every layout becomes a <div>:
HTML
<div class="video-preview">
  
  <img src="thumbnails/thumbnail-1.webp"> <div class="video-info-grid">
    
    <div class="channel-picture">
      <img src="profile.jpg">
    </div>


    <div class="video-info">
      <p class="video-title">My Video Title</p>
      <p class="video-author">Channel Name</p>
      <p class="video-stats">19M views</p>
    </div>


  </div>
</div>


To make Layer 2 horizontal, you could set the .channel-picture and .video-info divs to display: inline-block; and use vertical-align: top; to line them up. However, inline-block has some nasty alignment quirks and HTML spacing issues. There is a much better, modern way to do this.

Chapter 10: CSS Grid
10.1 Introduction to Grid
While inline-block works for putting things side-by-side, it is a rigid and easily broken method. A much better tool for creating horizontal layouts and full page grids is CSS Grid.
Grid completely transforms how elements are placed. Instead of elements dictating their own space, Grid allows you to define strict columns and rows on the container, and the elements simply snap into those columns.
To create a grid, you need two steps applied to the container <div>:
1. display: grid; (This activates the grid system).
2. grid-template-columns: (This defines how many columns exist and how wide they are).
10.2 Creating Columns and Rows
Let's look at the syntax:
CSS
.my-grid {
  display: grid;
  grid-template-columns: 100px 100px 200px;
}


This single line of code dictates that the container has exactly three columns. The first is 100px wide, the second is 100px wide, and the third is 200px wide.
If you put five <div> elements inside .my-grid, the first three will fill the top row. Because there are only three columns, Grid will automatically wrap the fourth and fifth elements onto a brand new row beneath them.
10.3 The Magic of the fr Unit
If you want to create a layout where elements stretch perfectly to fill the screen, pixel measurements won't work. We need the Fractional Unit (fr), which stands for "free space."
Look at our YouTube video info row (Layer 2 from the previous chapter). We have a profile picture on the left that needs to be exactly 50px wide, but we want the text on the right to stretch and fill all the remaining space.
CSS
.video-info-grid {
  display: grid;
  grid-template-columns: 50px 1fr;
}


The 1fr unit tells the browser: "Calculate whatever space is left over in the container, and give 100% of it to this column."
If we want to build the main YouTube homepage, where we have three videos sitting side-by-side taking up equal amounts of space, we use fr for all of them:
CSS
.video-grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
}


This creates three perfectly equal columns that will dynamically grow and shrink as the user resizes their web browser! (Note: If you write 1fr 2fr, the second column will take up twice as much free space as the first).
10.4 Column and Row Gaps
With inline-block, you had to use CSS margin to push elements away from each other. With CSS Grid, you don't need margins. Grid has built-in properties to create precise gutters between your rows and columns:
CSS
.video-grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  column-gap: 16px;
  row-gap: 40px;
}


* column-gap: 16px; perfectly spaces out the horizontal distance between our videos.
* row-gap: 40px; perfectly spaces out the vertical distance between rows of videos.
With just a few lines of code, CSS Grid solves some of the most frustrating layout problems in web development!
Chapter 11: Flexbox
11.1 Introduction to Flexbox vs. CSS Grid
While CSS Grid is incredible for laying out rigid, 2-dimensional structures (like a wall of video thumbnails), there is another layout system built for 1-dimensional, fluid structures (like a website's top navigation bar). This is called Flexbox.
To turn a <div> into a flexbox container, you simply apply one property:
CSS
.header {
  display: flex;
}


The Key Difference: In CSS Grid, you define the columns first, and elements snap into those rigid columns. In Flexbox, the layout is driven by the content. Elements will shrink, grow, and align themselves fluidly based on the space available. If you remove an element from a Flexbox, the remaining elements will organically shift to fill the void. This makes Flexbox the perfect tool for responsive navigation bars.
11.2 The flex-direction Property
By default, activating Flexbox places all child elements side-by-side in a row. You can manually control this direction:
* flex-direction: row; (The default horizontal layout).
* flex-direction: column; (Stacks elements vertically, one on top of the other).
11.3 The flex Property (Flexbox's version of fr)
Just like Grid uses 1fr to take up remaining free space, Flexbox uses the flex property.
Imagine a YouTube header divided into three sections: a Left Section (logo), a Middle Section (search bar), and a Right Section (user icons). We want the left and right sections to have fixed widths, but we want the middle search bar to stretch and fill whatever space is left.
CSS
.left-section {
  width: 150px;
}
.middle-section {
  flex: 1; /* Stretches to fill the remaining free space */
}
.right-section {
  width: 200px;
}


If you resize your browser window, the 150px and 200px sections will stay perfectly frozen in size, while the .middle-section dynamically stretches and shrinks like a rubber band.
11.4 Alignment with justify-content and align-items
Flexbox gives you incredible, effortless control over aligning elements horizontally and vertically.
Horizontal Alignment (justify-content): If you have three buttons inside a flexbox, you dictate where they sit horizontally:
* justify-content: start; (Packs everything to the left).
* justify-content: end; (Packs everything to the right).
* justify-content: center; (Packs everything into the exact center).
* justify-content: space-between; (Spreads elements out evenly, pushing the first item to the far left edge and the last item to the far right edge).
Vertical Alignment (align-items): By default, elements stretch from the top of a flexbox to the bottom (align-items: stretch). But if you have a 55px tall header and you want a small 24px icon to sit perfectly in the vertical middle, you use:
* align-items: center; (Perfectly centers elements vertically).
11.5 The flex-shrink Trick
Flexbox is sometimes too flexible. If you make your browser window extremely narrow, Flexbox will eventually start squishing the icons in your Right Section to make room.
To forcefully prevent a specific element from shrinking, you use flex-shrink: 0;.
CSS
.right-section {
  width: 200px;
  flex-shrink: 0; /* Tells the browser: "Never shrink this, no matter what!" */
}

Chapter 12: CSS Position
12.1 The Normal Flow (position: static)
By default, every HTML element has position: static;. This means they follow the normal rules of the page: they stack up, push each other down, and scroll away when you scroll down the page. To build professional UI elements—like a header that follows you as you scroll, or a tooltip that hovers over a button—we must break elements out of this normal flow.
12.2 Sticking to the Screen (position: fixed)
When you give an element position: fixed;, it completely detaches from the normal webpage and floats directly on your browser window.
CSS
.header {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  height: 55px;
  background-color: white;
}


* The Coordinates: We pin the header 0 pixels from the top, 0 pixels from the left, and 0 pixels from the right.
* The Background Color: Because the header is now floating above the page, it becomes transparent. You must manually add background-color: white; so the scrolling videos don't bleed through the header text.
* The Body Padding Fix: Because the header detached and floated upwards, the video grid below it instantly slammed up to the top of the page, hiding behind the header. To fix this, you must add padding-top: 80px; to your <body> to push your normal content safely below the fixed header.
(Note: We use this exact same position: fixed; trick to pin the sidebar to the left, bottom, and top of the screen!)
12.3 Stacking Order (z-index)
If you scroll down your YouTube clone, you might notice your video thumbnails overlapping and hiding your fixed header!
When elements detach from the normal flow, the browser has to guess which one belongs in front. By default, whichever element is written lowest in your HTML file appears in front. To manually override this, we use the z-index property.
CSS
.header {
  z-index: 100;
}


Elements with a higher z-index will always overlap elements with a lower z-index. Giving the header 100 guarantees it will securely glide over the top of the video grid.
12.4 The Golden Rule: Absolute Inside Relative
How do we place a small "14:20" timestamp badge precisely into the bottom-right corner of a video thumbnail? We use position: absolute;.
position: absolute; detaches an element and positions it based on coordinates, just like fixed. But instead of sticking to the browser window, it sticks to the page (meaning it will scroll away naturally).
However, there is a golden rule in CSS: If you place an absolute element inside a relative container, the absolute element will position itself based on the corners of that container, not the corners of the whole webpage!
HTML
<div class="thumbnail-container"> <img src="thumbnail.webp">
  <div class="video-time">14:20</div> </div>


CSS
.thumbnail-container {
  position: relative; /* Acts as the anchor */
}
.video-time {
  position: absolute;
  bottom: 8px; /* 8px from the bottom of the thumbnail */
  right: 8px;  /* 8px from the right of the thumbnail */
}


This is how we overlay text onto images, create notification badges (like a tiny "3" hovering over a bell icon), and create tooltips.
12.5 Creating Hover Tooltips
To create a tooltip that says "Search" when you hover over the magnifying glass button, we use the "Absolute inside Relative" trick to anchor a black text box directly beneath the button.
To hide it and make it appear only on hover, we manipulate the opacity:
CSS
/* The hidden tooltip */
.tooltip {
  position: absolute;
  bottom: -30px;
  opacity: 0; /* Invisible by default */
  transition: opacity 0.15s;
  pointer-events: none; /* Prevents the tooltip itself from triggering hover glitches */
}


/* When the button is hovered, change the tooltip's opacity to 1 */
.search-button:hover .tooltip {
  opacity: 1; 
}


Using an advanced CSS selector like .search-button:hover .tooltip, we tell the browser: "Only target the .tooltip class when my mouse is hovering over the .search-button container."
Chapter 13: Responsive Design & Final Polish
13.1 Responsive Design (Media Queries)
If you look at the real YouTube on a large desktop monitor, you might see four or five videos in a single row. If you look at it on a smaller laptop, you might see three. On a mobile phone, you will likely only see one or two. The layout adjusts to ensure the website always looks good, regardless of the screen size. This is called Responsive Design.
To implement responsive design, we use a CSS feature called a Media Query. A media query tells the browser to only activate a specific block of CSS code if the screen size matches a certain condition.
CSS
/* Default grid for standard screens (3 columns) */
.video-grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
}


/* For smaller screens: activate 2 columns */
@media (max-width: 750px) {
  .video-grid {
    grid-template-columns: 1fr 1fr;
  }
}


/* For large screens: activate 4 columns */
@media (min-width: 1000px) {
  .video-grid {
    grid-template-columns: 1fr 1fr 1fr 1fr;
  }
}


* max-width: 750px: This means the styles inside the curly braces will only apply if the screen is 750 pixels wide or smaller.
* min-width: 1000px: This means the styles will only apply if the screen is 1000 pixels wide or larger.
(Note: You can find your exact screen width in pixels by opening the Chrome DevTools and resizing your browser window; the dimensions will appear in the top right corner).
13.2 CSS Shorthand Properties
As your CSS file grows, you want to write less code. Instead of writing out every direction for padding or margins, you can use Shorthand Properties.
Instead of writing this:
CSS
padding-top: 10px;
padding-bottom: 10px;
padding-left: 20px;
padding-right: 20px;


You can write exactly the same thing in one line:
CSS
padding: 10px 20px;


* One value (padding: 4px;): Applies 4px to all four sides equally.
* Two values (padding: 10px 20px;): The first value is the vertical padding (top and bottom). The second value is the horizontal padding (left and right).
* Four values (padding: 10px 20px 30px 40px;): Applies padding in a clockwise circle starting from the top: Top, Right, Bottom, Left.
This also works for borders! Instead of writing border-width, border-style, and border-color on three separate lines, you can combine them:
CSS
border: 1px solid gray;


13.3 CSS Inheritance
Inheritance is a powerful feature where certain CSS properties applied to an outer container are automatically "passed down" to all the elements inside it.
Instead of setting font-family: Roboto, Arial; on the video title, the video author, the sidebar links, and the header, you can set it once on the very top-level element of your page: the <body> element.
CSS
body {
  font-family: Roboto, Arial;
  background-color: rgb(248, 248, 248);
}


Because every single visible element on your webpage lives inside the <body>, they will all inherit the Roboto font.
The Specificity Override: What if you set color: red; on the <body>, but you want your video author text to be gray? Inheritance will make everything red, but remember the rule of CSS Specificity from Chapter 5. A specific class selector (.video-author) has higher priority than a broad element selector (body).
CSS
.video-author {
  color: rgb(96, 96, 96); /* This overrides the inherited red color! */
}


13.4 Semantic HTML
Throughout the course, we used the <div> element as a generic container. While <div> works perfectly for layout and styling, it doesn't tell the browser what the content actually is.
To help search engines (like Google) and screen readers (for visually impaired users) understand your website, you should replace generic <div> tags with Semantic Elements where appropriate. They behave exactly like a <div>, but they have special meaning to robots:
* <header>: Used for the top navigation bar instead of <div class="header">.
* <nav>: Used for the sidebar instead of <div class="sidebar"> (stands for navigation).
* <main>: Wraps around the primary content of your webpage (the video grid).
* <section>: Used to group related content inside your main area.
13.5 Leaving Comments
When working on large projects or collaborating with a team, it is essential to leave notes in your code explaining why you did something. The computer completely ignores comments, meaning they won't affect your website.
HTML Comments: Enclose your text with ``.
HTML
<div class="left-section">...</div>


CSS Comments: Enclose your text with /* and */.
CSS
.tooltip {
  /* We use absolute positioning inside a relative container to overlay the tooltip */
  position: absolute; 
}
